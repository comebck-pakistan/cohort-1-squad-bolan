# Invoice 007 Prompt Comparison

## Objective

Compare three prompt designs using the same invoice.

---

# Prompt V2

### Characteristics

- Generic OCR
- Broad extraction instructions

### Result

- Multiple hallucinated fields
- Ignored invoice structure
- Failed to extract business information accurately

Overall Performance

Poor

---

# Prompt V4

### Characteristics

- Generic OCR
- Explicit anti-hallucination rules
- No guessing handwritten text

### Result

- Hallucinations removed
- Better handling of unreadable text
- More reliable OCR extraction

Limitations

- Missing invoice-specific structure
- Missing promotions
- Missing taxes
- Missing payment details

Overall Performance

Good

---

# Prompt V3

### Characteristics

- Invoice-specific schema
- Domain-specific field definitions
- Structured JSON output

### Result

- Supplier extracted correctly
- Customer extracted correctly
- Product rows extracted correctly
- Taxes extracted correctly
- Promotions extracted correctly
- Totals extracted correctly
- Payment information extracted correctly

Minor Issue

Only a small handwritten note was interpreted differently from manual inspection.

Overall Performance

Excellent

---

# Final Conclusion

The experiments demonstrate that prompt engineering significantly affects OCR quality.

Adding hallucination prevention improves generic OCR performance.

However, an invoice-specific prompt still provides substantially better structured extraction for distributor invoices.

For this project, Prompt V3 is recommended as the production prompt because it consistently produces complete, structured invoice data suitable for downstream processing.
